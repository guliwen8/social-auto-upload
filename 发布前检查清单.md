## 发布前检查清单（Release Checklist）

适用范围：本项目在你本机或服务器上准备“可稳定执行发布任务”前的最后检查。覆盖前端、平台服务、平台 worker、以及腾讯视频/YouTube 两条已验收链路。

### 0. 代码与版本

- 确认当前分支/提交为本次联调通过的版本（包含 tencent + youtube 的修复）。
- `git status` 处于干净状态（无未提交改动），或已明确哪些改动不需要提交。
- 如果要发版打 tag：先确保 CI/本机构建通过（见下文）。

### 1. Python 环境

- Python 版本在 `3.10 ~ 3.12`（项目声明 `>=3.10,<3.13`）。
- 建议使用项目 venv：
  - `.venv` 已创建并激活：`source .venv/bin/activate`
  - `pip install -r requirements.txt` 无报错
- Playwright / 浏览器依赖已装好：
  - `python -m playwright install`（基础组件）
  - `python -m playwright install chrome`（YouTube 必需的 Chrome channel）

### 2. Chrome 与代理（YouTube 重点）

- 机器上存在可用的 Chrome（Playwright 的 `channel="chrome"` 能启动）。
- 如需代理（直连 YouTube 不稳定/超时）：
  - 方式 A：环境变量 `YT_PROXY`（推荐，便于临时切换）
    - 示例：`export YT_PROXY="http://127.0.0.1:7890"`
  - 方式 B：`conf.py` 中的 `YT_PROXY`（已支持从环境变量覆盖）

### 3. 账号登录态（cookies）

账号文件默认在 `cookies/` 目录下。

- 腾讯视频（视频号助手）：
  - 账号文件存在：`cookies/tencent_<account_name>.json`
  - 校验命令：`python3 sau_cli.py tencent check --account <account_name>` 输出 `valid`
- YouTube：
  - 账号文件存在：`cookies/youtube_<account_name>.json`
  - 校验命令：`python3 sau_cli.py youtube check --account <account_name>` 输出 `valid`
  - 注意：YouTube 上传在 headless 下容易被 Studio 判为“不支持的浏览器”，上传实现已强制使用 headed 模式；运行任务时请允许浏览器窗口打开并保持到任务结束。

### 4. 前端（控制面）构建与基本可用性

- `sau_frontend` 能成功构建：在 `sau_frontend/` 执行 `npm run build` 成功。
- 控制面能看到 YouTube 平台选项与字段：
  - 账号页可创建 `youtube`
  - 发布页选择 YouTube 账号后显示：`可见性 / 播放列表 / 缩略图路径`

### 5. 平台服务与 worker

- 平台服务能启动并监听端口（示例）：
  - `python3 sau_platform_server.py --host 127.0.0.1 --port 5411`
- 如使用 worker（自动执行队列/定时）：
  - `python3 sau_platform_worker.py` 能启动且不报错
- 数据库可写：
  - `data/platform.db` 可读写（无权限错误）

### 6. 最小回归（发布前必跑一次）

每个平台至少跑一次“从控制面创建到任务成功回写”的最小闭环，保证不是只在 CLI 成功。

- 腾讯视频（视频号助手）最小回归：
  - 创建草稿（视频素材）
  - 创建计划（tencent）
  - 提交计划后执行任务
  - 期望：task 状态回写 `succeeded`
- YouTube 最小回归：
  - 创建草稿（视频素材）
  - 创建计划（youtube，建议先用 `unlisted`）
  - 提交计划后执行任务
  - 期望：task 状态回写 `succeeded`，并在日志中看到发布完成链接

### 7. 常见失败快速定位（必备）

- `patchright` 缺失：本机跑 `sau_cli.py` 直接报 `No module named patchright`
  - 处理：确认 venv + `pip install -r requirements.txt`
- YouTube 提示 Chrome 不存在：`Chromium distribution 'chrome' is not found`
  - 处理：`python -m playwright install chrome`
- YouTube upload 找不到上传框 / “unsupported browser”
  - 处理：确认不是 headless；必要时配置 `YT_PROXY`
- 任务被冷却/断路器拦截
  - 现状：控制面手动执行接口已允许 bypass（用于人工重试），自动调度仍保留保护

### 8. 发布/交付前建议（可选）

- 把验收总结归档到版本：
  - `tencent-acceptance/tencent-acceptance.html`
  - `youtube-acceptance/youtube-acceptance.html`
- 如果要交付给他人运行：
  - 明确要求 Python 版本、Chrome、代理方式、以及“首次必须交互式登录生成 cookies”
